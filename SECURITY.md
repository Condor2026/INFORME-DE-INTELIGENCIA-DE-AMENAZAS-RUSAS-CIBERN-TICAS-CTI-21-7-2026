# Política de Seguridad

Este repositorio contiene información crítica sobre infraestructura maliciosa activa (C2, Phishing, Malware). La seguridad de los datos y la integridad de los indicadores son nuestra prioridad.

## Reporte de Vulnerabilidades o Nuevas Amenazas

Si eres un investigador de seguridad o un usuario y has identificado:

- Nuevos dominios relacionados con `goo.su` o la IP `82.202.170.126`.
- Nuevos hashes de archivos maliciosos que se comunican con esta infraestructura.
- Nuevas cuentas en redes sociales (X, Facebook, etc.) distribuyendo estos enlaces.

Por favor, reporta esta información a través de los siguientes canales:

1. **Issues del Repositorio:** Abre un Issue con la etiqueta `threat-intel` y adjunta la evidencia (capturas de pantalla, logs, hashes).
2. **Contacto Directo:** Si el material es sensible, puedes contactar al autor vía [Correo Electrónico] (simulado) o a través de los mecanismos de seguridad de GitHub.

## Alcance de la Seguridad

Este repositorio **NO** contiene exploits activos ni código malicioso funcional. Contiene Indicadores de Compromiso (IOCs) y análisis forense.

**Advertencia:** El uso de estos IOCs para actividades no autorizadas (acceso a sistemas sin consentimiento) está estrictamente prohibido. Este material es exclusivamente para fines de **Defensa Cibernética**, **Threat Hunting** y **Educación**.

## Actualización de IOCs

Los IOCs contenidos en `iocs.csv` y `iocs.json` se actualizan a medida que se descubren nuevos artefactos. Se recomienda revisar este repositorio periódicamente para mantener actualizadas las listas de bloqueo en firewalls, SIEMs y EDRs.
